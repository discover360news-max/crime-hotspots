## Using Dashboard Filters

The dashboard has a Filters panel that lets you control exactly which incidents appear on the map and in the statistics.

## Opening the filters

On desktop, the Filters panel is pinned to the right side of the dashboard. On mobile, tap the **Filters** button at the bottom of the screen to open a slide-up panel.

## Available filters

### Year

Defaults to the current year. Use the year selector to switch to a previous year's data. The map, incident counts, and all statistics will update to reflect the selected year.

### Crime type

Filter by one or more crime types (e.g. show only Murders and Shootings). When no crime type is selected, all incident types are shown. Tap a crime type chip to toggle it.

### Date range

Use the date pickers to set a custom start and end date within the selected year. This is useful for analysing a specific period — for example, the Carnival weekend or a specific month.

### Area

Type in the area filter field to narrow the map to a single neighbourhood or community. The map will pan to that area and show only incidents from that location.

## Resetting filters

Tap **Clear Filters** to return all filters to their default state (current year, all crime types, full date range, no area restriction).

## Filters and the map

The map updates in real time as you change filters. Cluster bubbles show the incident count for each geographic cluster — tap a cluster to zoom in, or tap an individual pin to see the incident details.

## Quick Insights

Below the map, the Quick Insights panel shows automatically calculated summaries: highest crime area, peak day of week, most common crime type, and total incident count — all calculated for the current filter state.
